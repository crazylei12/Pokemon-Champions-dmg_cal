# 更新日志

本文件记录 Pokémon Champions Assistant 面向用户的版本变化。

## [1.1.4] - 2026-07-24

### 新增与改进

- 新增可长期保存的对手宝可梦配置。在对局中可把当前能力点、性格、特性和道具命名保存；首页“我保存的宝可梦配置”支持脱离对局新建、搜索、编辑和二次确认删除。
- 用户配置纳入整包备份与恢复，并提供只包含宝可梦配置的分享文件。导入会保留本机其他配置、按配置 ID 更新并保持重复导入幂等；损坏数据会阻止覆盖写入，并可在保留原始副本后由用户确认重置。
- 基础形态与对应 Mega 形态、以及 Champions 对战内可切换形态现在共享同组用户配置和本局临时调整；切换后按目标形态种族值重算能力，并在特性不兼容时安全回退。

### 伤害与输入修复

- 对手未知招式候选改为使用 Pokémon Champions 招式快照，不再混入游戏未收录招式；用户已经确认的我方实际招式优先于静态招式池，即使快照暂未收录也不会在伤害面板或形态切换时丢失。
- 首页和悬浮编辑界面的单行文本输入统一处理软键盘“完成”动作，避免完成键无响应或键盘无法正确收起。

### 兼容性与迁移

- 版本提升为 `1.1.4 (9)`，最低系统仍为 Android 13；录屏入口仍仅在 Android 16（API 36）开放。
- 从旧版升级会保留本机已有队伍、对局状态、HUD 布局、设置和对手配置；旧版整包备份不含对手配置字段时，也不会清空设备上已有配置。
- 标准版与录屏功能版继续使用相同包名、版本号和固定生产签名，可以互相覆盖但不能同时安装；公开 Release 仍只提供两个独立的 `arm64-v8a` APK。

完整说明见 [Android 1.1.4 发布说明](docs/android_1.1.4_release_notes_zh.md)。

## [1.1.3] - 2026-07-23

### 新增与改进

- 战斗 HUD 顶部新增单打 / 双打切换。单打仅显示左下方我方与右上方对方各一只及两只速度线，双打恢复原有四只布局；切换格式会同步关闭或默认开启范围招式伤害修正。

### 修复

- 将 Smogon damage-calc 内核更新到 `3677e41`，纳入 Pokémon Champions 能力下降等级的现代公式修复，避免特定速度线和依赖速度的伤害计算使用旧世代取整。

### 性能

- 进一步减少传统悬浮战斗面板在选人、耐久预设和战斗状态变化时的重复窗口更新、伤害重算与持久化写入，降低实战操作时的主线程负担。

### 兼容性与迁移

- 版本提升为 `1.1.3 (8)`，最低系统仍为 Android 13；录屏入口仍仅在 Android 16（API 36）开放。
- 标准版与录屏功能版继续使用相同包名、版本号和固定生产签名，可以互相覆盖并保留本地队伍、HUD 布局与设置，但不能同时安装。
- 公开 Release 继续只提供两个独立的 `arm64-v8a` APK；本地构建的 `x86_64` 产物仅用于模拟器验证。

完整说明见 [Android 1.1.3 发布说明](docs/android_1.1.3_release_notes_zh.md)。

## [1.1.2] - 2026-07-22

### 新增与改进

- 新增可选战斗 HUD 模式。确认双方阵容后可直接在游戏画面上查看四只宝可梦的速度顺序、四个招式的伤害区间，以及双方场上宝可梦和对方耐久预设。
- HUD 顶部提供布局调整、再战识别、显示/隐藏和己方队伍识别入口；录屏功能版还保留独立录像入口。HUD 模式运行时不再额外显示悬浮球。
- HUD 部件支持拖动和缩放，并分别保存横屏、竖屏布局；所有部件持续约束在当前屏幕安全区域内。
- README 新增手机实战 HUD 截图和简要使用说明。

### 修复与性能

- 悬浮伤害面板及条件、速度线、对手编辑子页现在可以收起并恢复上次页面；相关页面会共享拖动位置，不再在切换后跳回默认位置。
- 对方耐久预设改为按队伍槽位保存，切换目标再返回时恢复该宝可梦上次使用的预设。
- 减少 HUD 常规状态变化时的窗口重建、WebView 重算和重复写盘，提升实战更新稳定性。
- 修复版本与构建身份文件变化后 Gradle 配置缓存可能沿用旧值的问题；发布校验现在会直接比对 APK 包内版本号。

### 兼容性与迁移

- 版本提升为 `1.1.2 (7)`，最低系统仍为 Android 13；录屏入口仍仅在 Android 16（API 36）开放。
- 标准版与录屏功能版继续使用相同包名、版本号和固定生产签名，可以互相覆盖并保留本地队伍、HUD 布局与设置，但不能同时安装。
- 公开 Release 继续只提供两个独立的 `arm64-v8a` APK；本地构建的 `x86_64` 产物仅用于模拟器验证。

完整说明见 [Android 1.1.2 发布说明](docs/android_1.1.2_release_notes_zh.md)。

## [1.1.1] - 2026-07-17

### 新增与改进

- 同一个 GitHub Release 现在同时提供推荐的标准版和可选的录屏功能版，两个安装包均为 `arm64-v8a`，并通过明确的文件名帮助用户按需选择。
- 录屏功能版在 Android 16 上提供独立的开始/结束录屏入口，可把单应用游戏画面与允许捕获的游戏内部声音保存为本机 MP4；录屏开始、结束和失败不会关闭队伍识别或伤害面板。
- 应用内更新页会识别当前安装包身份：标准版默认选择标准资产，录屏功能版默认选择录屏资产，同时两边都显示另一个版本的切换入口。

### 修复

- 悬浮球、菜单、人工修正页和伤害面板会按当前可用屏幕安全区域重新定位，降低横竖屏切换、系统栏和折叠屏窗口变化后控件越界或无法操作的风险。
- 录屏功能版修复了 ColorOS 播放音频捕获重启失败、录屏资源收尾、静止画面读取和录屏/识别生命周期互相干扰等问题。

### 兼容性与迁移

- 版本提升为 `1.1.1 (6)`，最低系统仍为 Android 13；录屏入口首发仅在 Android 16（API 36）开放。
- 标准版与录屏功能版使用相同包名、版本号和固定生产签名，不能同时安装，但可互相覆盖并保留本地队伍与设置。
- 两个分支分别写入 `standard` / `replay` 构建身份。用户互相覆盖切换后，后续应用内检查更新的默认项会跟随当前实际安装版本。

完整说明见 [Android 1.1.1 发布说明](docs/android_1.1.1_release_notes_zh.md)。

## [1.1.0] - 2026-07-17

### 新增与改进

- 己方双页 OCR 识别完成后统一进入人工确认页，即使自动识别已经完整也不会直接跳到命名保存；只有 6 个槽位的必填内容全部确认后才能保存队伍。
- 人工确认页为 6 项实际能力值显示明确字段名，并按原始招式槽位保留空缺位置，方便准确定位需要补录的内容。
- OCR 数值改为综合多个裁剪候选的一致性、位数和像素形态，不再简单选择最大值；补充 `6/9`、`2/3` 混淆纠正及安全带。
- 宝可梦残缺名称只有在候选唯一时才自动采用；招式识别增加排除属性图标的纯文字区域重试，降低跨物种误配和首招漏识别。
- 新增明确的 `android:assemble-release-arm64` 发布入口，本次只编译并发布手机版 `arm64-v8a` APK，不生成模拟器或 universal 安装包。

### 修复

- 修复 Android 单应用截图在横竖屏切换后沿用错误缓冲区方向，导致识别画面旋转或裁剪异常的问题。
- 修复悬浮窗焦点、触摸转发和展开/收起状态不安全，恢复己方修正页与实战伤害面板在手机和平板上的可拖动、可点击布局。
- 修复手机样本中的能力值候选偏大、`6/9` 误读、整卡 OCR 兜底绕过位数校验、残缺名称误匹配和招式属性图标污染。
- 修复平板样本中“大嘴娃—嬉闹”漏识别、耿鬼生命 `137` 被识别为 `127`，以及第二页不完整结果绕过人工修正的问题。
- 修复己方修正页空能力值输入框缺少标签，并统一对局、识别时机和游戏内队伍导航文案。

### 兼容性与迁移

- 最低系统仍为 Android 13（API 33），公开安装包仅支持 64 位 ARM（`arm64-v8a`）。
- `v1.1.0` 继续使用 `v1.0.1` 启用的固定生产签名；已安装 `v1.0.1` 的设备可以直接覆盖升级并保留本地队伍与设置，无需数据迁移。
- 保存规则现在要求每只宝可梦的 4 个招式槽位及其他必填字段全部确认，旧版允许 1–3 个招式的草稿需在确认页补全后再保存。

### 验证

- `npm.cmd test`、Android 单元测试、release lint、第三方许可证检查和依赖安全审计全部通过。
- 手机版 Release 构建只生成一个 `arm64-v8a` APK；版本、唯一生产签名、核心识别特征包/ROI 哈希、许可证资产与更新专用网络权限校验通过。
- 修复过程已在 RMX3820 手机和 OPD2409 平板的真实双页样本上复测，覆盖完整导入、缺字段人工修正与原有保存队伍保留。

完整发布说明见 [Android 1.1.0 发布说明](docs/android_1.1.0_release_notes_zh.md)，OCR 根因与实机证据见 [己方队伍 OCR 根因与修复记录](docs/android_own_team_ocr_issue_root_cause_and_fix_2026-07-16_zh.md)。

## [1.0.1] - 2026-07-15

### 新增与改进

- 将双方队伍预览识别所需的 `team-preview-templates-v2.bin` 及校验元数据作为核心运行资源纳入源码并强制打入 APK；离线安装后即可直接使用，发布检查会验证文件大小、二进制哈希与 ROI 哈希。
- 每次重新识别双方队伍都创建一场新比赛；旧局的槽位、烧伤、能力等级、天气、场地和现场覆盖全部重置，并要求用户重新确认后才能进入伤害面板。
- 对战状态改为按双方每个槽位分别保存；同局切换己方已保存队伍时只重置己方槽位状态，保留对手和公共场地状态。
- 对手道具支持“沿用预设”“明确无道具”“指定实际道具”三态；队伍只要拥有 1–3 个有效招式即可保存和计算，缺少关键特性或能力值时会明确阻止计算。
- 新增不含截图的 JSON 整包导出与恢复，并启用 Android 加密备份/设备迁移规则。
- 强化正式签名、单 ABI、许可证、网络权限和识别特征包的发布门禁；正式 Release 只提供 `arm64-v8a` APK，`x86_64` 仅保留给本地模拟器。

### 修复

- 修复双方队伍识别和己方双页 OCR 在关闭服务时先释放识别器、仍有后台任务访问已关闭对象的竞态，避免崩溃、线程池拒绝、幽灵确认窗口和销毁后写入 UI/文件。
- 修复己方 OCR 延迟取帧在服务结束后仍可能执行并覆盖“会话已结束”状态的问题；未分发的取帧任务现在可取消，销毁后的回调不会再更新界面。
- 修复不同宝可梦槽位之间烧伤、能力等级等现场状态可能串位的问题。
- 修复不完整队伍仍可进入计算、重复招式计数和部分手动修正流程不一致的问题。

### 兼容性与迁移

- 最低系统版本为 Android 13（API 33），公开安装包仅支持 64 位 ARM（`arm64-v8a`）。
- `v1.0.1` 启用了新的固定生产签名。已经安装 `v1.0.0` 的设备若系统提示签名不一致，应先导出需要保留的数据，再卸载旧版并安装 `v1.0.1`；从 `v1.0.1` 起后续正式版本将继续使用同一生产证书。

### 验证

- OPPO OPD2409 / Android 16 实机相册环境完成 14 项验收：12 项通过、1 项部分通过、1 项按要求免验，已知 P0 发布阻断已解除。
- Node 回归 10/10、Android 单元测试 41/41、release lint、第三方许可证检查、依赖安全审计以及两个单 ABI APK 发布校验均通过。

完整发布说明见 [Android 1.0.1 发布说明](docs/android_1.0.1_release_notes_zh.md)，详细实机证据见 [Android 1.0.1 相册环境验收结果](docs/android_1.0.1_gallery_acceptance_result_2026-07-15_zh.md)。
