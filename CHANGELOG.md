# 更新日志

本文件记录 Pokémon Champions Assistant 面向用户的版本变化。

## [1.0.1] - 2026-07-15

### 新增与改进

- 将双方队伍预览识别所需的 `team-preview-templates-v2.bin` 及校验元数据作为核心运行资源纳入源码并强制打入 APK；离线安装后即可直接使用，发布检查会验证文件大小、二进制哈希与 ROI 哈希。
- 每次重新识别双方队伍都创建一场新比赛；旧局的槽位、烧伤、能力等级、天气、场地和现场覆盖全部重置，并要求用户重新确认后才能进入伤害面板。
- 对战状态改为按双方每个槽位分别保存；同局切换己方已保存队伍时只重置己方槽位状态，保留对手和公共场地状态。
- 对手道具支持“沿用预设”“明确无道具”“指定实际道具”三态；队伍只要拥有 1–3 个有效招式即可保存和计算，缺少关键特性或能力值时会明确阻止计算。
- 新增不含截图的 JSON 整包导出与恢复，并启用 Android 加密备份/设备迁移规则。
- 强化正式签名、单 ABI、许可证、网络权限和识别特征包的发布门禁；正式 Release 只提供 `arm64-v8a` APK，`x86_64` 仅保留给本地模拟器。

### 修复

- 修复双方队伍识别和己方双页 OCR 在关闭服务时先释放识别器、仍有后台任务访问已关闭对象的竞态，避免崩溃、线程池拒绝、幽灵确认窗口和销毁后写入 UI/文件。
- 修复己方 OCR 延迟取帧在服务结束后仍可能执行并覆盖“会话已结束”状态的问题；未分发的取帧任务现在可取消，销毁后的回调不会再更新界面。
- 修复不同宝可梦槽位之间烧伤、能力等级等现场状态可能串位的问题。
- 修复不完整队伍仍可进入计算、重复招式计数和部分手动修正流程不一致的问题。

### 兼容性与迁移

- 最低系统版本为 Android 13（API 33），公开安装包仅支持 64 位 ARM（`arm64-v8a`）。
- `v1.0.1` 启用了新的固定生产签名。已经安装 `v1.0.0` 的设备若系统提示签名不一致，应先导出需要保留的数据，再卸载旧版并安装 `v1.0.1`；从 `v1.0.1` 起后续正式版本将继续使用同一生产证书。

### 验证

- OPPO OPD2409 / Android 16 实机相册环境完成 14 项验收：12 项通过、1 项部分通过、1 项按要求免验，已知 P0 发布阻断已解除。
- Node 回归 10/10、Android 单元测试 41/41、release lint、第三方许可证检查、依赖安全审计以及两个单 ABI APK 发布校验均通过。

完整发布说明见 [Android 1.0.1 发布说明](docs/android_1.0.1_release_notes_zh.md)，详细实机证据见 [Android 1.0.1 相册环境验收结果](docs/android_1.0.1_gallery_acceptance_result_2026-07-15_zh.md)。

## [1.0.0] - 2026-07-15

- 首个稳定正式版本，提供本地双向伤害计算、队伍管理、己方双页 OCR、悬浮对战面板以及用户主动检查更新。
- 该版本未随 APK 提供双方队伍预览识别特征包；此限制已在 `v1.0.1` 解除。
