# Android 1.0.1 发布说明

发布日期：2026-07-15

版本：`1.0.1 (4)`

发布标签：`v1.0.1`

## 下载与系统要求

- 正式安装包：`Pokemon-Champions-Assistant-v1.0.1-arm64.apk`
- 支持架构：仅 `arm64-v8a`
- 最低系统：Android 13（API 33）
- APK 大小：70,693,215 字节
- APK SHA-256：`B0DD01CE7B82C6DDC363907B571A0D394701B86CDADB3BA0A267B2BA769E3876`
- 生产签名证书 SHA-256：`671B45190A9DAC81A2747355CB9F10703503F1302EAF3E59582A282DD827EEF8`

本次 GitHub Release 只上传真机使用的 ARM64 APK。项目仍会构建并验证 `x86_64` 单 ABI APK，但它只用于本地 Android Studio 模拟器，不作为公开下载项；项目不生成 universal、32 位 ARM 或 x86 APK。

## 主要变化

1. 双方队伍识别现在开箱即用。1016 条模板组成的 `team-preview-templates-v2.bin` 特征包及其校验元数据已经作为核心运行资源进入源码和 APK，断网也可识别，不再依赖用户机器上的额外文件。
2. 每次重新识别双方队伍都视为一场新比赛。旧局的双方槽位、烧伤、能力等级、天气、场地和现场覆盖会被清空，只有重新确认识别结果后才能打开新局伤害面板。
3. 对局状态按双方每个槽位隔离；同一局内切换己方已保存队伍时，只重置己方槽位状态，保留对手状态与公共场地。
4. 对手道具支持沿用预设、明确无道具、指定实际道具三态。1–3 个有效招式也可以保存和计算；关键特性或实际能力值缺失时会给出明确原因并阻止错误计算。
5. 设置页新增不含截图的 JSON 整包导出/恢复，并启用 Android 加密备份和设备迁移规则。
6. 发布流程新增固定生产签名、单 ABI、许可证、更新专用网络权限及核心特征包/ROI 哈希硬校验。

## 关键修复

- 修复双方队伍识别和己方双页 OCR 的服务关闭竞态。服务停止后会先拒绝新任务，等待已经接收的识别任务安全结束，再释放 OpenCV/ML Kit 识别器和工作线程；销毁后的回调不会继续修改 UI、状态或文件。
- 修复己方 OCR 的延迟取帧任务在服务结束后仍可能执行、把“悬浮识别会话已结束”覆盖成旧进度或错误提示的问题。
- 修复不同宝可梦槽位之间烧伤、能力等级等状态可能串位的问题。
- 修复不完整队伍可误进入计算、重复招式计数以及部分手动修正不一致的问题。

## 从 1.0.0 更新

`v1.0.1` 是固定生产签名启用后的首个版本。若设备已经安装 `v1.0.0`，Android 可能因旧版签名不同而拒绝覆盖安装：

1. 如旧版中有需要保留的数据，先在设置页导出整包；
2. 卸载 `v1.0.0`；
3. 安装 `v1.0.1`；
4. 需要时恢复刚才导出的整包。

从 `v1.0.1` 起，后续正式版本会继续使用同一生产证书并递增 `versionCode`，以支持保留私有数据的覆盖升级。

## 已知说明

- OPD2409 横屏时，如果把悬浮面板拖到屏幕最上方，右上角“返回/收起”可能与 ColorOS 状态栏点击区域重叠；将悬浮面板向下拖离状态栏即可正常操作。这不会影响识别或伤害计算。
- 屏幕截图与 OCR 依赖 Android 的 MediaProjection 授权。App 或会话被系统终止后，需要用户重新授权；App 不会在后台静默恢复该权限。
