# Android 对局回放 Phase 4 实现与验收记录

日期：2026-07-17

分支：`feature/battle-replay-phase-4`

状态：Phase 4 实现和 RMX3820 单应用验收已完成。最终验收按用户要求只在游戏大厅进行，不进入正式对战。

## 1. 本阶段交付

Phase 4 开放 `识别并录屏`，继续复用 Phase 1～3 已有的单应用授权、唯一 `VirtualDisplay`、H.264/AAC 编码、MediaStore 发布和统一停止入口。组合模式没有创建第二套投屏，也没有把 OCR 改成 540p 输入。

```text
Pokemon Champions 单应用 MediaProjection
  -> 唯一 VirtualDisplay
  -> OES SurfaceTexture
  -> EglProjectionRouter
       -> 持续 960 x 540 / 24 fps H.264 编码
       -> 用户请求时渲染原捕获尺寸 FBO
       -> glReadPixels RGBA
       -> 独立线程做上下翻转和 RGBA -> Android ARGB
       -> 现有队伍识别、人工核对和伤害面板
  -> 游戏 UID PCM -> 48 kHz AAC
  -> MP4
```

### 1.1 原分辨率按需读回

- `EglProjectionRouter` 新增原捕获尺寸纹理和 FBO，只在用户点击识别相关入口时创建、渲染和读回。
- `glReadPixels` 留在 EGL 线程；RGBA 转 ARGB、GL 底部原点到 Bitmap 顶部原点的逐行转换放到独立 `HandlerThread`。
- 读取结果带 `glReadbackMs`、`conversionMs` 和总耗时，便于实机定位卡顿。
- 同一时刻只允许一个读回请求；取消、超时、转换失败和路由关闭都会回调失败，不停止视频或音频编码。
- `onCapturedContentResize()` 后使静态帧缓存失效，避免用旧尺寸/旧方向读取。
- 对完全静止的游戏画面缓存最近一次 `SurfaceTexture` 变换矩阵，点击悬浮球后可立即读取当前纹理，不再无条件等待下一次 `onFrameAvailable()`。RMX3820 的静态维护画面已验证该修复。

### 1.2 组合模式和生命周期

- Android 16 的模式菜单正式启用 `识别并录屏`。
- 组合模式先准备识别宿主，再用同一个 replay recorder 创建唯一捕获 Surface。
- 菜单打开前先冻结一张原分辨率游戏帧；之后点击“录入我的队伍”或“识别双方阵容”时复用该帧，避免把菜单打开后的时机误当成识别画面。
- 如果冻结失败，菜单仍可打开；实际点击识别时再尝试一次读回。失败提示明确说明录屏仍继续。
- 识别完成回调只有在会话仍为 `RUNNING` 时才能保存结果或打开核对页；结束服务、取消或新一代请求会使旧回调失效。
- `结束并保存 MP4`、通知停止、系统停止仍走同一个幂等 `requestStop()`；停止时同时取消尚未完成的识别读回。

### 1.3 调试验收增强

Debug `ReplayArtifactVerifier` 从原先首/中/尾三帧扩为五个时间线检查点：开始、1/4、1/2、2/3 和接近结尾。这样可以把悬浮菜单和伤害面板实际展开的时间段直接抽成帧，而不是只依赖现场截图。

## 3. RMX3820 实机结果

设备：RMX3820，Android 16 / API 36，ADB 序列号 `6465e08`。

### 3.1 原分辨率识别集成

- 组合模式实际输入为 `2772 x 1240`，编码画布仍为 `960 x 540`。
- 一次实机原分辨率帧复制耗时约 `506.958 ms`，随后双方阵容识别约 `2639 ms`；期间录制时长继续增加，伤害面板可正常打开。
- 静态画面上第一次实现会等待下一帧而超时；增加最近纹理矩阵缓存后，同一静态画面点击悬浮球可立即打开菜单并读取当前帧。
- 实战 TEAM_PREVIEW 上组合链路能够进入核对页，但该样本的宝可梦候选准确率不理想。它属于现有识别数据/模型质量问题，正式对战中的准确率和人工修正由用户后续手动验收；本阶段没有用错误候选冒充识别正确。

### 3.2 按用户指定方式进行的最终录制

用户要求停止自动进入正式对战，最终验收改为：

1. 回到 Pokémon Champions 大厅；
2. 选择系统授权页中的 Pokémon Champions 单应用；
3. 开始 `识别并录屏`，游戏内部声音预检通过；
4. 录制过程中展开悬浮菜单；
5. 在大厅直接打开已有会话的伤害面板，保持一段时间后收起；
6. 从悬浮菜单选择“结束并保存 MP4”；
7. 对同一 MediaStore 文件做轨道、时间戳和五点解码检查。
