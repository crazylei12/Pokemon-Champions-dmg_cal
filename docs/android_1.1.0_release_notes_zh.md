# Android 1.1.0 发布说明

发布日期：2026-07-17

版本：`1.1.0 (5)`

发布标签：`v1.1.0`

## 下载与系统要求

- 正式安装包：`Pokemon-Champions-Assistant-v1.1.0-arm64.apk`
- 支持架构：仅 `arm64-v8a`
- 最低系统：Android 13（API 33）
- APK 大小：`71,056,740` 字节
- APK SHA-256：`38E80B0169E6BB1E9CB4C411F4E3CC2330F92D49D0BEA5AB01F5FABD083E2EDE`
- 生产签名证书 SHA-256：`671B45190A9DAC81A2747355CB9F10703503F1302EAF3E59582A282DD827EEF8`

本次按照手机版发布边界只编译并上传一个 ARM64 APK，没有生成 `x86_64`、universal、32 位 ARM 或 x86 安装包。

## 主要变化

1. 己方队伍双页 OCR 现在始终进入人工确认页。即使自动识别完整，也必须由用户检查 6 个槽位后再命名保存，避免高置信度误读被静默写入队伍。
2. 修正页会明确标出 HP、攻击、防御、特攻、特防和速度；缺失招式按原始槽位显示，用户可以直接定位未识别字段。
3. 能力值 OCR 会综合多个裁剪候选的一致性、位数和数字像素形态，并对 `6/9`、`2/3` 易混淆数字使用带安全区的纠正规则。
4. 残缺宝可梦名称不再按目录顺序猜测；只有候选唯一时才自动采用。首招低置信度时会使用排除属性图标的纯文字区域重试。
5. 对局首页、双方队伍预览、己方双页识别和游戏内队伍导航文案重新梳理，明确“何时截图、何时切页、何时确认”。

## 关键修复

- 修复“单个应用”MediaProjection 在横竖屏切换后缓冲区方向错误，导致截图旋转或识别裁剪异常的问题。
- 修复悬浮面板焦点和触摸转发冲突，恢复手机与平板上的拖动、展开、输入、返回和收起操作。
- 修复多种真机 OCR 边界：数值候选误选最大值、雷丘 `99` 的 `6/9` 混淆、奇麒麟 `90` 被整卡兜底拼成 `900`、残缺中文名跨物种冲突，以及招式属性图标污染文字。
- 修复平板样本中“大嘴娃—嬉闹”漏识别、耿鬼生命 `137 → 127`，以及能力值页缺一项时错误绕过人工修正的问题。
- 修复空白实际能力值输入框没有字段名、用户无法判断应补录哪项能力的问题。

## 从 1.0.1 更新

`v1.1.0` 沿用 `v1.0.1` 的固定生产证书，并把 Android `versionCode` 从 4 提升到 5。已安装 `v1.0.1` 的设备可以直接覆盖安装，App 私有目录中的保存队伍、设置和导出能力不需要迁移。

保存校验比旧版更严格：每只宝可梦必须确认 4 个招式槽位以及其他必填字段。旧版中只有 1–3 个招式的未完成草稿会进入人工确认页，补全后才能保存；这不会修改已经保存的完整队伍。

- Node/TypeScript 回归：`10/10` 通过。
- Android 单元测试：`54/54` 通过，失败、错误和跳过均为 0。
- release lint、第三方许可证一致性检查和 `npm.cmd audit --omit=dev --audit-level=high` 通过。
- 构建目录中只存在一个 `arm64-v8a` Release APK；APK 内版本为 `1.1.0 (5)`，只有一个生产签名者，证书指纹与 `v1.0.1` 一致。
- APK 发布校验确认：唯一原生 ABI 为 `arm64-v8a`，核心识别特征包与 ROI 哈希匹配，中文/拉丁 ML Kit 许可证资产完整，仅保留用户主动检查更新所需的网络权限。
- 己方队伍 OCR 修复已在 RMX3820 手机与 OPD2409 平板的真实双页样本上完成复测；自动识别、缺字段修正、命名保存和原有队伍保留路径均有记录。

详细根因、样本结果和设备侧证据见 [己方队伍 OCR 根因与修复记录](https://github.com/crazylei12/Pokemon-Champions-dmg_cal/blob/v1.1.0/docs/android_own_team_ocr_issue_root_cause_and_fix_2026-07-16_zh.md)。

## 已知说明

- 屏幕截图与 OCR 依赖 Android 的 MediaProjection 授权。App、共享目标或会话被系统终止后，需要用户重新授权；App 不会在后台静默恢复该权限。
- 自动识别结果始终是待确认草稿。画面处于动画、非全屏或系统/相册控件压缩内容时，仍应等待画面稳定后重试或手动修正。
- 本版本只提供 64 位 ARM 安装包，不支持 x86_64 模拟器直接安装此 Release 资产。
