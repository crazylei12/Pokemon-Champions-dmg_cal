## 标准版变化

- 悬浮球、菜单、人工修正页和伤害面板会持续按当前屏幕安全区域约束位置，降低横竖屏切换、系统栏或窗口形态变化后控件越界的问题。
- 应用内更新页会同时识别标准与录屏 APK，并按当前安装包的构建身份排序：标准版优先显示标准资产，录屏功能版优先显示录屏资产。
- 继续包含 1.1.0 的己方双页 OCR 人工确认、能力值纠错、完整招式槽位校验和双方队伍预览识别能力。

## 录屏功能版额外能力

- Android 16 上可从悬浮菜单独立开始或结束录屏，不需要在“识别”和“录屏”之间切换模式。
- 使用 Android 单应用 MediaProjection 捕获 Pokémon Champions，保存 H.264 MP4 到本机系统媒体库；助手悬浮球、菜单、核对页和伤害面板不会写入成品画面。
- 在系统和游戏允许时捕获游戏内部声音；检测不到可用声音时，只能由用户明确取消或选择无声视频，不会改录麦克风。
- 录屏与识别、人工核对和伤害面板分离；结束录屏并保存 MP4 后，对局助手可以继续运行。
- 修复 ColorOS 设备上声音预检后正式音频采集无法重启、失败资源未完整回收、静止画面无法及时读回等问题。

## 更新与版本选择

- 从 `v1.1.0` 升级时，两个 APK 都沿用固定生产签名并把 Android `versionCode` 从 5 提升到 6，可直接覆盖并保留本地数据。
- 标准版内“检查更新”默认选择标准版，录屏功能版内默认选择录屏功能版；两个页面都同时提供另一个版本，用户可以主动互相切换。
- 两个分支分别打入 `standard` / `replay` 构建身份。用户覆盖安装切换版本后，下一次检查更新的默认项会跟随当前实际安装的 APK。

## 已知说明

- 录屏功能版首发只在 Android 16 开放；Android 13～15 安装该 APK 后仍可使用标准的识别和伤害功能，但菜单不会显示录屏入口。
- 游戏内部声音能否被捕获受 Android、设备厂商实现和游戏播放策略共同约束。检测失败时应用不会偷偷使用麦克风，用户可取消或明确选择无声录制。
- 必须在系统共享界面选择 Pokémon Champions 单个应用；录屏功能版不是整机通用录屏器，也不会绕过 Android 的投屏授权与内容保护。
- 自动识别结果仍是待确认草稿。画面处于动画、非全屏或系统控件压缩内容时，应等待画面稳定后重试或手动修正。
