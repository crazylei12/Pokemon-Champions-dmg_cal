## 单打与双打 HUD

- 战斗 HUD 顶部新增“单打 / 双打”切换，不需要退出当前 HUD 或重新识别队伍。
- 单打布局只显示左下方我方与右上方对方各一只场上宝可梦，并把速度线收敛为两只；双打恢复双方各两只的原有布局。
- 切换到单打会关闭范围招式伤害修正，切回双打会默认开启；双方 6 只队伍的识别、切换和耐久预设仍保留。
- HUD 的目标选择、招式伤害区间、速度顺序、戏法空间反转、布局拖动与横竖屏保存方式保持不变。

## 伤害计算修复

- Smogon damage-calc 内核更新到 `3677e41`，纳入 Pokémon Champions 能力下降等级的现代公式修复。
- 修复特定负能力等级下速度线和依赖速度的伤害计算可能使用旧世代取整的问题；Android 离线伤害引擎与 TypeScript 计算入口已同步更新。

## 传统悬浮面板性能

- 选人、切换对方耐久预设、改变战斗状态或刷新速度线时，面板会更精确地判断哪些内容实际发生变化。
- 减少无效窗口更新、重复伤害重算和持久化写入，降低实战连续操作时的主线程负担。
- 这项优化不改变原有伤害结果、对局状态含义、子页恢复规则或用户保存的数据格式。

## 录屏功能版额外能力

- Android 16 上可从 HUD 顶部或悬浮菜单独立开始、停止录屏，把 Pokémon Champions 单应用画面与系统允许捕获的游戏内部声音保存为本机 H.264 MP4。
- 助手的 HUD、悬浮球、菜单、核对页和伤害面板不会写入成品画面；录屏与识别、人工核对和伤害计算可以继续并行使用。
- 检测不到可用游戏声音时，应用只允许用户取消或明确选择无声视频，不会改录麦克风。

## 更新与兼容性

- 从 `v1.1.2` 升级时，两个 APK 都沿用固定生产签名并把 Android `versionCode` 从 7 提升到 8，可直接覆盖并保留本地数据。
- 标准版内“检查更新”默认选择标准资产，录屏功能版默认选择录屏资产；两个页面都同时提供另一个版本的切换入口。
- 最低系统仍为 Android 13（API 33）。录屏入口仍只在 Android 16（API 36）开放；Android 13～15 安装录屏功能版后可使用识别、伤害面板与 HUD，但不会显示录像入口。

## 已知说明

- 自动识别结果仍是待确认草稿。画面处于动画、非全屏或系统控件压缩内容时，应等待画面稳定后重试或手动修正。
- 自动化测试和 APK 校验不能替代真实对局中的触控、布局与帧时间验收；不同尺寸、横竖屏和 OEM 窗口行为仍应按设备实际结果判断。
- 游戏内部声音能否被捕获受 Android、设备厂商实现和游戏播放策略共同约束。录屏功能版不是整机通用录屏器，也不会绕过 Android 的投屏授权与内容保护。
