## 保存和管理对手配置

- 在“调整对手配置”中可把当前能力点、性格、特性和道具命名保存为长期配置，后续对局可直接复用。
- 首页新增“我保存的宝可梦配置”管理页，可按宝可梦或配置名称搜索，也可不进入对局，手动选择宝可梦或具体形态后新建空白配置。
- 配置支持修改名称、能力点、性格、特性和道具；删除前需要二次确认。若删除当前对局正在使用的配置，对局会清除失效引用并回退到仍然存在的配置。
- 用户配置统一显示在内置模板之前，并按全局保存顺序排列。

## 分享、备份和损坏保护

- 设置页“导出/导入我的数据”的整包迁移现在始终包含用户保存的宝可梦配置，并在结果中报告配置数量。
- 新增只包含用户宝可梦配置的分享文件。导入时保留本机其他配置，相同配置 ID 更新，新 ID 按文件顺序追加；重复导入同一文件不会生成副本。
- 从旧版本生成、完全没有用户配置字段的整包备份继续兼容，恢复时不会误删本机已有配置；字段存在但结构无效时会拒绝导入。
- 本机用户配置文件损坏时，App 不会把它静默当成空配置继续覆盖。内置模板仍可使用，但修改和导出会被阻止；用户确认重置前会先保留原始损坏文件副本。

## 形态间配置继承

- 基础形态与对应 Mega 形态共享同组用户配置和本局临时调整。
- 坚盾剑怪、飘浮泡泡、谜拟Ｑ、莫鲁贝可和海豚侠等可在单场对战内切换的形态也共享配置。
- 切换到目标形态时保留能力点、性格、道具和已确认招式，按目标形态种族值重新计算能力值；原特性在目标形态不可用时回退为目标形态默认特性。
- 地区形态、洛托姆家电形态、大小、花纹、性别等其他多形态继续相互隔离，避免错误套用配置。

## 招式候选和输入修复

- 对手未知招式候选改为使用 Pokémon Champions 招式快照，不再从更大的通用资料中混入当前游戏未收录招式。
- 我方队伍中的招式是用户已经确认的实际状态，优先于静态招式池；即使 Champions 快照暂未收录某招式，切换到伤害面板或切换形态时也不会丢失。
- 首页和悬浮编辑界面的单行文本输入统一支持软键盘“完成”动作，并在提交后正确收起键盘。

## 录屏功能版额外能力

- Android 16 上可从 HUD 顶部或悬浮菜单独立开始、停止录屏，把 Pokémon Champions 单应用画面与系统允许捕获的游戏内部声音保存为本机 H.264 MP4。
- 助手的 HUD、悬浮球、菜单、核对页和伤害面板不会写入成品画面；录屏与识别、人工核对和伤害计算可以继续并行使用。
- 检测不到可用游戏声音时，应用只允许用户取消或明确选择无声视频，不会改录麦克风。

## 更新与兼容性

- 从 `v1.1.3` 升级时，两个 APK 都沿用固定生产签名并把 Android `versionCode` 从 8 提升到 9，可直接覆盖并保留本地数据。
- 标准版内“检查更新”默认选择标准资产，录屏功能版默认选择录屏资产；两个页面都同时提供另一个版本的切换入口。
- 最低系统仍为 Android 13（API 33）。录屏入口仍只在 Android 16（API 36）开放；Android 13～15 安装录屏功能版后可使用识别、伤害面板、对手配置与 HUD，但不会显示录像入口。

## 已知说明

- 自动识别结果仍是待确认草稿。画面处于动画、非全屏或系统控件压缩内容时，应等待画面稳定后重试或手动修正。
- 用户保存的对手配置表达战术假设，不代表对手真实配置；导入分享文件前仍应检查名称、能力点、性格、特性和道具。
- 自动化测试和 APK 校验不能替代真实对局中的配置编辑、形态切换、导入导出和覆盖升级验收；不同尺寸、横竖屏和 OEM 窗口行为仍应按设备实际结果判断。
- 游戏内部声音能否被捕获受 Android、设备厂商实现和游戏播放策略共同约束。录屏功能版不是整机通用录屏器，也不会绕过 Android 的投屏授权与内容保护。
