# Android 1.1.2 发布说明

发布日期：2026-07-22

版本：`1.1.2 (7)`

正式版标签：`v1.1.2`

录屏功能版构建分支：`feature/battle-replay-phase-4`

## 下载与选择

本次 Release 提供两个只能二选一安装的 ARM64 APK：

| 版本 | 文件 | 适合用户 | 系统要求 |
| --- | --- | --- | --- |
| 标准版（推荐） | `Pokemon-Champions-Assistant-v1.1.2-arm64.apk` | 队伍识别、伤害计算、传统悬浮面板和战斗 HUD | Android 13+ |
| 录屏功能版（可选） | `Pokemon-Champions-Assistant-v1.1.2-replay-arm64.apk` | 在标准功能之外，还需要把对局保存为本机 MP4 | Android 16 |

- 标准版大小：`STANDARD_APK_BYTES` 字节
- 标准版 SHA-256：`STANDARD_APK_SHA256`
- 录屏功能版大小：`REPLAY_APK_BYTES` 字节
- 录屏功能版 SHA-256：`REPLAY_APK_SHA256`
- 生产签名证书 SHA-256：`671B45190A9DAC81A2747355CB9F10703503F1302EAF3E59582A282DD827EEF8`
- 两个公开 APK 的唯一原生 ABI 均为 `arm64-v8a`；Release 不提供 universal、32 位 ARM、x86 或 x86_64 安装包。

两个版本使用相同的应用包名、`versionCode=7` 和生产签名，因此不能同时安装。标准版与录屏功能版可以互相覆盖，并保留 App 私有目录中的队伍、对局状态、HUD 布局与设置；重要数据仍建议先使用设置页的整包导出功能备份。

## 战斗 HUD

- “对局”页新增独立的 HUD 启动模式；双方阵容确认完成后直接显示 HUD，不再额外保留悬浮球。普通模式保持原有悬浮球与完整面板流程。
- HUD 同时展示场上四只宝可梦的速度点或范围、先手顺序，以及我方当前宝可梦四个招式对目标的伤害区间。戏法空间开启后顺序会自动反转。
- HUD 可直接切换双方当前场上宝可梦和对方耐久预设；每只对方宝可梦会记住自己的预设，切换回来无需重新选择。
- 顶部“调整”进入布局编辑，可拖动部件或拖动右下角改变大小；横屏、竖屏布局分别保存，并持续限制在当前安全区域内。
- “隐藏 HUD”临时收起主要信息，“再战”开始新一局双方阵容识别，“识别我方”录入或更新自己的保存队伍。
- README 已加入 RMX3820 / Android 16 的实战 HUD 示例图，说明四只速度顺序、伤害区间、目标选择和快捷控制的对应位置。

## 传统悬浮面板改进

- 伤害面板、条件、速度线和对手编辑子页可以收起并在重新打开时恢复上次页面；“返回”仍回到主伤害页，重新识别双方阵容会重置到主页面。
- 相关子页共享拖动后的屏幕位置，但各自保留尺寸、滚动位置和页面内容，减少切换时的跳动。
- HUD 的常规选人、预设和状态变化会尽量复用现有窗口与缓存，减少重复重建和伤害引擎计算。

## 录屏功能版额外能力

- Android 16 上可从 HUD 顶部或悬浮菜单独立开始、停止录屏，把 Pokémon Champions 单应用画面与系统允许捕获的游戏内部声音保存为本机 H.264 MP4。
- 助手的 HUD、悬浮球、菜单、核对页和伤害面板不会写入成品画面；录屏与识别、人工核对和伤害计算可以继续并行使用。
- 检测不到可用游戏声音时，应用只允许用户取消或明确选择无声视频，不会改录麦克风。

## 更新与兼容性

- 从 `v1.1.1` 升级时，两个 APK 都沿用固定生产签名并把 Android `versionCode` 从 6 提升到 7，可直接覆盖并保留本地数据。
- 标准版内“检查更新”默认选择标准资产，录屏功能版默认选择录屏资产；两个页面都同时提供另一个版本的切换入口。
- 最低系统仍为 Android 13（API 33）。录屏入口仍只在 Android 16（API 36）开放；Android 13～15 安装录屏功能版后可使用识别、伤害面板与 HUD，但不会显示录像入口。

## 验证结果

- 标准版：`STANDARD_TEST_SUMMARY`。
- 录屏功能版：`REPLAY_TEST_SUMMARY`。
- 两个分支均构建并验证了独立的 `arm64-v8a` 与 `x86_64` 单 ABI Release APK；GitHub Release 只上传两个 ARM64 成品。
- 两个 ARM64 APK 均通过版本、唯一生产签名、构建身份、核心识别特征包/ROI 哈希、许可证资产和权限边界检查。
- 标准版构建提交：`STANDARD_BUILD_COMMIT`；录屏功能版构建提交：`REPLAY_BUILD_COMMIT`。

## 已知说明

- 自动识别结果仍是待确认草稿。画面处于动画、非全屏或系统控件压缩内容时，应等待画面稳定后重试或手动修正。
- HUD 会显示在用户当前设备画面上，但 Android 的单应用 MediaProjection 会把助手窗口从录屏成品中隔离；不同 OEM 的窗口与音频行为仍应按设备实际结果判断。
- 游戏内部声音能否被捕获受 Android、设备厂商实现和游戏播放策略共同约束。录屏功能版不是整机通用录屏器，也不会绕过 Android 的投屏授权与内容保护。
